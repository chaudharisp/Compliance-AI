How the Fallback System Works

1. Priority Order

Every question first tries Gemini 2.5 Pro (best quality, 25 requests/day). When those 25 are used up, it automatically switches to Gemini 2.5 Flash (1,500/day). If that's exhausted too, it falls back to Flash-Lite (3,000/day).

2. Automatic & Transparent

You don't need to do anything — the system switches models instantly. The chat UI shows which model answered each question. The switch happens on rate limit errors (HTTP 429) from the Gemini API.

3. Combined Quota

Total daily capacity: 4,525 requests across all models. That's 25 Pro + 1,500 Flash + 3,000 Flash-Lite. All quotas reset at 12:00 AM Pacific Time.

Understanding Tokens

What are Input Tokens?

  • Your question — the text you type
  • Retrieved context — HIPAA/GDPR chunks the system finds relevant (typically 5 chunks × ~500 tokens each)
  • System prompt — instructions that tell the model to be a compliance assistant
  • Chat history — last 3 Q&A pairs if follow-up question

Typical question: ~2,500-4,000 input tokens

What are Output Tokens?

  • The AI's response text — everything it generates
  • Longer answers = more output tokens
  • Each response is capped at 65,536 tokens (~50K words) — more than enough
  • No daily output cap on the free tier — only input tokens and requests are limited

Typical answer: ~200-800 output tokens

How Tokens are Counted

  • Roughly 1 token ≈ 4 characters in English
  • 100 words ≈ ~75 tokens
  • A full page of text ≈ ~500-700 tokens
  • Our tracker estimates tokens locally (not billed counts)

Which Limits Actually Matter?

Requests Per Day (RPD) — Primary Limit

This is the main constraint. Gemini 2.5 Pro gives you 25 questions/day for free, Flash gives 1,500, and Lite gives 3,000. Our fallback system means you get 4,525 total.

Requests Per Minute (RPM) — Burst Limit

Prevents too many rapid requests. Pro allows 5/min, Flash 15/min, Lite 30/min. Usually not an issue for manual chat use.

Input Tokens Per Day (TPD) — Rarely Hit

Pro: 1M/day, Flash: 1.5M/day, Lite: 1.5M/day. At ~3K tokens per question, you'd need 333+ questions on Pro to hit this. RPD is always exhausted first.

Output Tokens — No Daily Limit

The free tier does not have a daily output token cap. Each response can be up to 65K tokens. You'll never hit an output limit in normal use.
